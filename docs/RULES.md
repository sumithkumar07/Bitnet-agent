# Sovereign Development Constitution (v1.0)

This repository is governed by a strict set of technical rules designed to ensure mathematical sincerity, hardware transparency, and agentic independence.

## ⚖️ Core Rules

1.  **Master Before Move (Numerical Verification):** Every phase must be numerically verified before proceeding. Every "Council" must prove its resonance.
2.  **Brutal Technical Sincerity:** No marketing jargon. Use accurate engineering nomenclature.
3.  **Minimal Engineering:** No "just in case" code. Only build what is required for the current phase's verification.
4.  **Hardware Proximity:** Prefer raw pointers, AVX2 intrinsics, and direct heap management over abstraction layers.
5.  **0.00% Zero-Corruption:** Information transfer between agents must be lossless or neurally-gated; never corrupted by overflow.
6.  **Self-Organizing Swarm:** The engine must prioritize the autonomy of the agent nodes over a centralized global control.
7.  **1.58-bit Integrity:** All weights and activations must converge to the ternary `{-1, 0, 1}` space. 
8.  **Atomic Persistence:** Any state frozen to disk must be bit-identical or bit-quantized during reconstitution. (Rule Added Phase 31).
9.  **Decentralized Metabolism:** Agents must hydrate and purge their own neural data (JIT Paging). (Rule Added Phase 33).
10. **Smallest Move Efficiency:** Prefer the smallest possible architectural change that achieves the goal.
11. **Agent-First Architecture:** Prioritize features that enhance "Agentic Intelligence" (Mitosis, Trade, Parallel Councils) over standard LLM bloat. (Rule Added Phase 30).
